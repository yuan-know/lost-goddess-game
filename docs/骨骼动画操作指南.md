# 骨骼动画操作指南 —— 老年形态 idle / walk

> 本指南覆盖**必须在 Unity 编辑器里手工完成**的部分(绑骨骼、蒙皮、摆关键帧、接线)。
> 能自动化的部分(装包、PSD 分层导入配置、Animator 状态机)已由程序做成菜单一键工具,
> 本指南会告诉你在哪一步点它们。
>
> 目标:让老年老人从"静止立绘"变成"待机时微动、行走时迈腿摆臂"的骨骼动画,
> 并接到 `PlayerController`,使其点击行走时自动播 walk、停下播 idle。
>
> 术语:**Bone**=骨骼,**Skinning/Weight**=蒙皮/权重(把图层像素绑到骨头上),
> **Rig**=骨架,**Clip**=动画片段。

---

## 阶段 0:环境就绪(程序已做,你只需确认)

1. **包已装**:`Packages/manifest.json` 已加 `com.unity.2d.psdimporter`。
   切回 Unity 后它会自动还原。确认:菜单 `Window ▸ Package Manager` → 左上下拉 `In Project`
   → 能看到 **2D PSD Importer** 和 **2D Animation**。
2. **PSD 已在项目里**:`Assets/_Project/Art/Characters/characters_3forms.psd`
   (含青/中/老三组,每组 12 个部位层 + 1 个完整立绘层)。

> ⚠️ 如果 Package Manager 里没有 2D PSD Importer:说明包没还原成功。
> 检查 manifest.json 有这行 `"com.unity.2d.psdimporter": "8.1.1"`,
> 然后 `Assets ▸ Reimport All` 或重启 Unity。

---

## 阶段 1:把 PSD 配成 Mosaic 骨骼导入(程序菜单,一键)

1. 点菜单 **`失落的女神 ▸ 配置角色PSD(Mosaic骨骼导入)`**。
   - 它会把 PSD 设成:Mosaic(各图层拆成独立 Sprite)+ Import Hidden(导入隐藏层,
     **本 PSD 部位层默认隐藏,必须开否则导进来是空的**)+ Character 模式(可绑骨骼)+
     PPU 600 + 锚点脚底,然后自动 Reimport。
2. 成功后,在 **Project 窗口点开 characters_3forms.psd 左边的小三角**,应能看到一堆子 Sprite:
   `老年头脖子`、`老年躯干大衣`、`老年右大臂肘关节`…… 每组 12 个,共 3 组。

> 💡 如果弹窗提示"PSD Importer 未生效 / 找不到 PSD",按弹窗说明处理(多半是包没还原,回阶段 0)。

> 💡 **只做老年**:PSD 里三组都会被导入。骨骼动画本次只处理老年那 12 个部位,
> 青/中年的 Sprite 先不管(它们不影响老年绑骨骼)。

---

## 阶段 2:进入 Skinning Editor 绑骨骼(手工核心)

1. Project 里**选中 characters_3forms.psd**(选整个 psd,不是子 Sprite)。
2. Inspector 里点 **`Sprite Editor`** 按钮 → 打开 Sprite Editor 窗口。
3. 窗口**左上角有个下拉**(默认写 `Sprite Editor`),点开切到 **`Skinning Editor`**。
   - 这时能看到画布上拼成完整人形的老人(各部位 Sprite 按 PSD 原位摆好)。

### 2.1 造骨骼(Create Bone)

左侧工具栏(Skinning Editor 的图标):
1. 点 **`Create Bone`**(骨头图标)。
2. 从**盆骨/腰部**开始画一根根骨头(点一下起点,再点终点定一根;继续点会接着上一根往下延伸,形成父子链)。
   老年人形推荐骨架(拄拐、佝偻,可简化):
   ```
   Root(盆骨,放在两腿之间靠上)
   ├─ Spine 脊柱(向上到胸)
   │   ├─ Head 头(脖子→头)
   │   ├─ Arm_L_Upper 左大臂 → Arm_L_Lower 左小臂手
   │   └─ Arm_R_Upper 右大臂 → Arm_R_Lower 右小臂手
   ├─ Leg_L_Upper 左大腿 → Leg_L_Lower 左小腿 → Foot_L 左脚
   └─ Leg_R_Upper 右大腿 → Leg_R_Lower 右小腿 → Foot_R 右脚
   ```
   - 画骨头时**要接着关节位置画**(肩、肘、髋、膝、踝),这样后面弯曲才自然。
   - 想另起一条不连续的骨链:先按 `Esc` 断开,再重新点起点。
3. 画错了用 **`Edit Bone`** 拖动调整位置/长度,或选中骨头删除重画。

### 2.2 蒙皮(把图层像素绑到骨头)

1. 点 **`Auto Geometry`**(自动生成网格):
   - 通常点 **`Generate For All Visible`**,让每个部位 Sprite 生成三角网格。
   - Outline Detail / Alpha Tolerance 用默认即可,Subdivide 可给 0~2。
2. 点 **`Auto Weights`**(自动权重):
   - **`Generate For All Visible`** —— Unity 自动把每块网格权重分配给最近的骨头。
3. **检查权重**:点 `Weight Preview`(或直接在 `Pose` 模式下拖动骨头),看部位是否跟着对的骨头动。
   - 手臂动时躯干不该跟着变形;头不该被腿的骨头拉扯。
   - 有问题用 **`Weight Brush`**(权重笔刷)在出问题的区域补画:选中目标骨头,
     在该部位上涂,增/减该骨头对这些像素的影响。

### 2.3 摆一个静止基准姿势(可选但推荐)

- 在 `Preview Pose` 里把老人摆成**自然拄拐站立**(略佝偻),作为 idle 的基准。
- 摆好后,Skinning Editor 顶部 **`Apply`** 保存 rig。**关掉 Sprite Editor 窗口时记得 Apply**,
  否则骨骼/权重不保存。

---

## 阶段 3:把角色拖进场景,准备做动画

1. 把 **characters_3forms.psd 从 Project 拖进 Hierarchy**(或拖进 Scene)。
   - 因为开了 `Generate GO Hierarchy`,它会生成一个带完整骨骼层级的 GameObject
     (根下有 Sprite Renderer 们 + Sprite Skin + 骨骼 Transform)。
   - 这个对象里**三组青/中/老都在**;把青年、中年那两组子物体先 `SetActive(false)` 或删掉,
     只留老年,重命名为 `OldMan_Rigged`。
2. 做成 Prefab:拖回 `Assets/_Project/Prefabs/`,命名 `OldMan.prefab`。

> 💡 若拖进去发现部位错位/散开:多半是 Auto Geometry/Weights 没做全,回阶段 2 补。

---

## 阶段 4:生成 Animator 状态机(程序菜单,一键)

1. 点菜单 **`失落的女神 ▸ 生成老年Animator脚手架`**。
   - 生成 `Assets/_Project/Art/Animations/Old/OldMan.controller`,内含:
     - 参数 **isWalking**(bool)—— 与 `PlayerController.walkParam` 一致
     - 状态 **Idle**(默认)/ **Walk**,已连好双向过渡
     - 占位 clip **Old_Idle**(不循环)/ **Old_Walk**(循环)
2. 之后你只需给这两个占位 clip 摆帧(阶段 5),不用自己连状态机。

---

## 阶段 5:摆 idle / walk 关键帧(手工核心)

1. 选中场景里的 `OldMan_Rigged`(或双击 Prefab 进入编辑)。
2. 给它加 **Animator 组件**(如果 PSD 生成的对象没自带):`Add Component ▸ Animator`,
   把 **Controller** 拖成 `OldMan.controller`。
3. 打开 **`Window ▸ Animation ▸ Animation`** 窗口(不是 Animator!Animation 是摆帧的)。
   - 窗口左上角的 clip 下拉,选 **Old_Idle**。

### 5.1 摆 Idle(待机微动 + 手抖)

- Idle 建议 1~2 秒循环。老年设定:**待机时手微微颤抖**,身体轻微起伏。
- 在 Animation 窗口点 **录制键(红点)**,把时间轴移到 0 帧:
  - 选中骨骼(在 Scene 里点骨头 Transform),此帧记录它们当前角度 → 打一个关键帧。
- 移到中间帧(如 0.5s):把**手/小臂骨头**旋转极小角度(1~3°),身体略微下沉 → 自动记帧。
- 末帧回到与 0 帧相同姿势(循环平滑)。
- 关掉录制。Old_Idle 已经是循环(程序设好了 loopTime)。

### 5.2 摆 Walk(拖拽步态)

- 切到 clip 下拉的 **Old_Walk**。老年设定:**擦地拖拽的缓慢步态**,幅度小。
- 一个最简 4 帧循环(0 / 0.25 / 0.5 / 0.75s,总长 ~1s):
  - 0s:左腿在前(略抬)、右腿在后;对侧手臂前后摆(幅度要小,老年迟缓)。
  - 0.25s:双腿接近并拢的过渡。
  - 0.5s:右腿在前、左腿在后(与 0s 镜像)。
  - 0.75s:并拢过渡。
  - (循环回 0s)
- 拄拐的话:拐杖(如果单独是个骨头/部位)随左手小幅前后点地。
- 身体整体略微上下起伏(佝偻,起伏比青年小)。

> 💡 摆帧技巧:先只摆大腿/小腿两三根骨头把"迈步"做出来,跑起来看感觉,再加手臂和身体起伏。
> 别一次调 12 根骨头。

---

## 阶段 6:接线到 PlayerController(手工,几下点击)

现在动画能播了,但要让"点击行走 → 自动切 walk/idle",需要把 Animator 接到 PlayerController。

**方式 A:直接用 OldMan.prefab 当玩家(推荐先验证)**
1. 给 `OldMan_Rigged` 加 **PlayerController** 组件(如果它还没有)。
2. Inspector 里 PlayerController:
   - **Animator** 字段 → 拖入本对象的 Animator 组件。
   - **Walk Param** → 保持 `isWalking`(默认值,别改)。
   - **Sprite Faces Right** → 老年立绘朝左,设 **false**(与沙盒一致)。
   - **Sorting Target** → 拖入躯干的 SpriteRenderer(或给根加 SortingGroup 后拖它)。
   - moveSpeed 老年建议 **2.2**。
3. Play:点空地老人走 → 应自动播 Old_Walk;到达停下 → 回 Old_Idle。

**方式 B:接到现有沙盒(SandboxBootstrap 目前是加载整张立绘 PNG)**
- 沙盒当前用的是 `Resources/Characters/old.png` 整张立绘(无骨骼)。
- 要让沙盒改用骨骼 Prefab,需要程序改 `SandboxBootstrap.BuildPlayer`
  改成实例化 `OldMan.prefab` 而非贴 PNG。**这一步告诉程序(A),我来改**,
  你先用方式 A 在一个空场景里验证骨骼动画本身跑通即可。

- [ ] Skinning Editor 里骨骼绑好、Pose 拖动骨头人物正确跟随变形
- [ ] Old_Idle 播放时老人待机微动/手抖,循环平滑

## 常见问题

| 现象 | 原因 / 解决 |
|------|------------|
| PSD 展开没有子 Sprite | 没配 Mosaic / Import Hidden。点菜单「配置角色PSD」。 |
| 导进来部位是空的/缺件 | Import Hidden 没开(本 PSD 部位层默认隐藏)。同上重配。 |
| 拖进场景部位散开错位 | Auto Geometry/Weights 没做全,回阶段 2.2。 |
| 拖骨头某部位不跟随 | 该部位权重没分到骨头,用 Weight Brush 补,或重跑 Auto Weights。 |
| 关窗口后骨骼没了 | 没点 Apply。Skinning Editor 改动必须 Apply 才保存。 |
| Play 不切 walk 动画 | PlayerController.animator 没拖、或 walkParam 不是 isWalking、或 Controller 没挂。 |
| 动画播了但人物不移动 | 位移由 PlayerController 代码驱动,不要在 walk clip 里 K 根节点位移(否则冲突)。 |

> **黄金法则**:位移(走多远)= PlayerController 代码负责;姿态(腿怎么迈)= 动画负责。
> walk clip 里**只摆骨头的相对旋转/摆动,不要移动角色根节点位置**。
