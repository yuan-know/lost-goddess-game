## 1. GameState —— 全局状态(存档核心)  🟡
所有进度、开关、物品都存这里。B 不要自己造存档。
```csharp
public static class GameState {
    // 布尔开关:门是否开、某物是否拿过、某谜题是否解开
    static bool GetFlag(string key);
    static void SetFlag(string key, bool value);

    // 物品
    static void AddItem(string itemId);
    static bool HasItem(string itemId);
    static void RemoveItem(string itemId);

    // 时间循环用:当前所处时代(序章先埋,后续启用)
    enum Era { Young, Middle, Old }
    static Era CurrentEra { get; }
}
```

## 2. InteractableBase —— 可交互物件基类  🟡
B 的所有可点击物继承它。

**交互流程(重要,因为本作主角可行走):**
玩家点击可交互物 → 系统让老人走到该物的 `interactPoint` → **到位后**才回调 `OnClick()`。
B 只需写 `OnClick()` 里"到位后发生什么",行走由系统自动处理。
若某物件不需要走近(远处的门、天上的月亮),把 `walkToBeforeInteract = false`,则点击立即触发。

```csharp
public abstract class InteractableBase : MonoBehaviour {
    // ── B 在 Inspector 配置 ──
    public Transform interactPoint;          // 老人交互前该站的位置(空物体);为空则站原地
    public bool walkToBeforeInteract = true; // 是否需要先走过去;false=点了立即触发

    // 必须实现:老人到位后发生什么(开抽屉、进特写、捡道具…)
    public abstract void OnClick();

    // 可用的受保护辅助:
    protected bool RequireItem(string id);   // 检查是否持有某道具(选中态见 §8 背包)
    protected void PlaySfx(string clip);     // 播音效(走 AudioManager)
    protected void Highlight(bool on);       // 鼠标悬停高亮
}
```
> 系统内部:点击命中一个 Interactable 时,若 `walkToBeforeInteract` 为真,先调 `PlayerController.WalkTo(interactPoint)` 并注册到达回调,到达后再执行 `OnClick()`;否则直接 `OnClick()`。B 不用管这套调度。

# 主角操控系统(点击寻路行走)

> **本作主角(老人)在场景内可行走。** 操控模型 = 光标点击 + 老人走到目标 + 视点/特写切换 + 背包。
> 寻路方案:**单屏可走带(直线移动 + 限制在可走区,不绕障碍)**;个别房间需要绕障碍时再单独升级为路点图。
> 玩家操控的四层:①点击交互 ②老人行走(玩家的"移动") ③房间/特写视点切换 ④背包选中→点目标。

## 7. PlayerController —— 老人行走控制  🟡
挂在老人 GameObject 上,每个房间场景放一个老人实例(或由系统在切场景后生成/定位)。B 一般不用直接调,交互系统会自动驱动它;仅剧情脚本可能需要"让老人走到某处"。
```csharp
public class PlayerController : MonoBehaviour {
    public float moveSpeed = 3f;             // 移动速度(单位/秒)

    // 走到某个世界坐标(点击空地时系统调用),到达后回调 onArrive
    void WalkTo(Vector2 worldPos, System.Action onArrive = null);
    // 走到某个 Transform(交互物的 interactPoint)
    void WalkTo(Transform target, System.Action onArrive = null);

    void StopMoving();                       // 立即停下(过场/对白时锁定)
    void SetControllable(bool on);           // 关闭时点击不再驱动行走(过场用)
    bool IsMoving { get; }

    // 内部行为(B 不用管):朝向翻转、idle/walk 动画切换、按脚底 Y 值动态深度排序、
    // 目标点被 WalkableArea clamp 到可走区内。
}
```

## 8. WalkableArea —— 房间可走区  🟡
每个房间场景挂一个,定义老人脚底能踩的范围。点击空地时,系统把目标点 clamp 进这个区域,老人不会走出画面/穿墙。
```csharp
public class WalkableArea : MonoBehaviour {
    // 用 PolygonCollider2D / 一条带状区域定义可走范围(在 Inspector 里画)
    Vector2 ClampToArea(Vector2 worldPos);   // 把任意点拉回最近的可走位置
    bool Contains(Vector2 worldPos);
}
```
> B 做房间时的义务:每个房间摆一个 WalkableArea 圈出地面;每个可交互物摆一个 `interactPoint` 空物体标记"老人站哪儿点它"。

## 9. InventorySystem —— 背包(选中→点目标)  🟡
物品使用手感:**在背包选中一件物品 → 光标进入"持物状态" → 点场景里的目标物**。目标物在 `OnClick()` 里用 `Inventory.SelectedItem` / `RequireItem` 判断。
```csharp
public static class InventorySystem {
    static void Add(string itemId);
    static void Remove(string itemId);
    static bool Has(string itemId);

    static string SelectedItem { get; }      // 当前选中的物品 id;无则为 null/空
    static void Select(string itemId);       // 玩家点背包槽时选中
    static void ClearSelection();            // 用完/取消

    static event System.Action<string> OnSelectionChanged; // UI 用:更新光标图标
}
```
> 交互物判定示例:`if (InventorySystem.SelectedItem == Keys.item_key) { 开锁; InventorySystem.Remove(...); }`

## 10. CloseupView —— 特写视图  🟡
点抽屉/信件等"细看"物件时,不换场景,而是弹出一个特写层(放大的子画面),里面可继续点交互,点返回退出。绣湖类大量谜题发生在特写里,避免每个抽屉开一个新 .unity。
```csharp
public static class CloseupView {
    static void Open(string closeupId);      // 打开某特写(预制体/子画面)
    static void Open(GameObject closeupPrefab);
    static void Close();                     // 返回房间
    static bool IsOpen { get; }
}
```
> 特写打开时,`PlayerController` 自动锁定(不响应行走点击),关闭后恢复。
